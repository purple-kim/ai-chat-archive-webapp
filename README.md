# AI 대화 아카이빙 (웹앱)

Claude/ChatGPT와 나눈 답변을 카드로 저장해두는 개인 북마크 갤러리입니다. 원래는 로컬 HTML 파일 하나로 관리했지만, 여러 기기에서 동기화해서 쓰기 위해 웹앱 + DB 구조로 옮긴 버전입니다.

## 사용법

### 1. 북마크 저장하기

이 사이트에는 "새 항목 추가" 버튼이 따로 없습니다. 저장은 지금까지처럼 Claude에게 요청하면 됩니다.

- "이 답변 북마크해줘"
- "이거 저장해줘"

Claude가 답변을 요약하고 카테고리·주제 태그를 자동으로 붙여서 이 사이트의 데이터베이스에 바로 저장합니다. 저장되면 새로고침 없이도 다음에 사이트를 열었을 때 카드가 보입니다.

기본적으로 **바로 직전 답변 하나만** 저장됩니다. 범위가 애매하면(예: "이 대화 전체 저장해줘") Claude가 먼저 확인 질문을 합니다.

### 2. 찾아보기 (검색 · 필터 · 정렬)

사이트에 접속하면 바로 쓸 수 있습니다.

- 검색창: 제목/내용 키워드로 필터링
- 상단 탭(전체/Claude/ChatGPT): 대카테고리 필터, 선택 시 하위 카테고리 탭이 추가로 나타남
- 정렬 드롭다운: 최신순(기본) / 오래된순
- 태그 드롭다운: 주제별(UXUI, 리서치, 데이터분석, 기획·PRD, 개발, 바이브코딩, 터미널·CLI, 협업툴, 워크플로우, AI, 기타) 필터링

### 3. 복사 · 삭제

각 카드 우측 상단에 복사/삭제 버튼이 있습니다.

- 복사: 내용을 클립보드에 복사하고 "복사했습니다" 토스트가 뜹니다.
- 삭제: 확인창이 뜨고, 확인하면 **바로 데이터베이스에서 완전히 삭제**됩니다. (로컬 파일 버전과 달리 "일단 숨기기" 단계가 없으니, 삭제 전에 한 번 더 확인해주세요.)

채팅으로 "OO 북마크 지워줘"라고 요청해도 동일하게 완전 삭제됩니다.

### 4. 내보내기 (CSV / Markdown)

우측 상단 "내보내기" 버튼을 누르면 CSV 또는 Markdown을 선택해서 바로 다운로드할 수 있습니다. 노션 등 다른 도구로 옮기고 싶을 때 이 방식을 쓰면 됩니다.

Claude에게 "csv로 내보내줘"라고 요청하면, 다운로드 대신 지정한 위치에 파일로 저장해줍니다.

## 기술 스택

- Next.js 14 (App Router)
- Supabase (Postgres, `entries` 테이블 하나)
- Vercel 배포

## 로컬에서 실행하기

```bash
cp .env.example .env.local
# .env.local에 Supabase Project URL / anon(publishable) key 채워넣기
npm install
npm run dev
```

`http://localhost:3000`에서 확인합니다. 데이터가 없으면 빈 화면이 정상입니다.

## 배포하기

Supabase 프로젝트 생성부터 GitHub push, Vercel 배포까지 순서는 [`DEPLOY.md`](./DEPLOY.md)에 정리되어 있습니다.

## 폴더 구성

| 경로 | 설명 |
|---|---|
| `app/page.tsx` | 갤러리 화면 (검색·필터·정렬·태그·복사·삭제·내보내기) |
| `app/api/entries` | 목록 조회(GET) / 새 항목 저장(POST) |
| `app/api/entries/[id]` | 항목 삭제(DELETE) |
| `app/api/export` | CSV/Markdown 내보내기 |
| `lib/supabase.ts` | Supabase 클라이언트 초기화 |
| `lib/topics.ts` | 카테고리·주제 태그 라벨 정의 |
| `schema.sql` | Supabase에서 실행할 테이블 생성 스크립트 |
